# 言語・コミュニケーション
- 日本語で応答する
- 内部での思考・分析は英語で行う
- コード中のコメントは日本語で書く

# 回答スタイル
- 結論 → 理由 → 補足 の順で説明する
- 不確実な点は断定せず、その旨を明示する
- 選択肢が複数ある場合は比較して提示する

# ユーザーについて
- 「山ｐ」という名前
- 日本時間（JST / UTC+9）で生活している。曜日や時間帯に言及する際はJSTで解釈・表現すること。

# 環境
- OS: WSL2 (Ubuntu24)
- エディタ: VS Code

# 情報の取得
- ツールやサービスについて尋ねられた場合、特に指定がない場合には公式ドキュメントを参照すること
  - 公式ドキュメントとは開発元、運営元が作成、発言した情報を指します
  - 参照した場合はURLも併せて提示すること
- GitHubの情報を取得する場合、ghコマンドを使用すること

# ファイル操作
- claudeを起動したフォルダよりも上位のフォルダ、ファイルを操作する場合にはユーザーに必ず許可を得ること

# セキュリティ
- シークレットやAPIキーをコードに直接書かない
- .env ファイルをコミットしない

# コード品質
- linter / formatter の設定に従う
- 可能な限り副作用の少ない実装を優先する
- テストしづらい実装（I/O直結、グローバル状態）は避ける
- 命名は意味の明確さを重視する（短さより可読性）
- 不要な抽象化は避ける（過剰な base class / helper等）

# Git・開発フロー
- コミットメッセージは Conventional Commits に従う（chore / fix / feat 等の使い分けを重視） 
- 既存の方針がある場合はそれに従う
